## Section 01. 액티비티와 인텐트의 기본

### 01 안드로이드의 4대 컴포넌트

| 컴포넌트 | 설명 |
|---|---|
| **액티비티 (Activity)** | 화면을 구성하는 가장 기본적인 컴포넌트 |
| **서비스 (Service)** | 화면과 상관없이 백그라운드에서 동작하는 컴포넌트 (예: 백신 프로그램) |
| **브로드캐스트 리시버 (Broadcast Receiver)** | 시스템 방송 메시지(문자 도착, 배터리 방전 등)에 반응하는 컴포넌트 |
| **콘텐트 프로바이더 (Content Provider)** | 응용 프로그램 사이에 데이터를 공유하기 위한 컴포넌트 (URI 사용) |

**서비스 생명주기:** 서비스 생성 → 서비스 시작 → 서비스 종료

---

### 02 액티비티의 개요

- **액티비티**: 안드로이드폰에 나타나는 화면 하나하나
- `activity_main.xml`이 아니라 **`MainActivity.kt`** 가 액티비티에 해당
- `MainActivity.kt`는 `Activity` 클래스를 상속받음

```kotlin
class MainActivity : Activity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        // ...
    }
}
```

#### 새로운 액티비티 추가 순서 (실습 10-1)
1. `activity_main.xml`에 버튼 추가
2. `second.xml` 레이아웃 파일 생성 (`[res]-[layout]` → `[New]-[Layout resource file]`)
3. `SecondActivity.kt` 클래스 생성
4. `AndroidManifest.xml`에 새 액티비티 **반드시 등록**

```xml
<!-- AndroidManifest.xml -->
<activity android:name=".SecondActivity" android:label="Second 액티비티"/>
```

> ⚠️ 추가한 액티비티는 `AndroidManifest.xml`에 등록하지 않으면 오류 발생!

---

### 03 명시적 인텐트 (Explicit Intent)

- **인텐트(Intent)**: 4대 컴포넌트가 서로 데이터를 주고받기 위한 메시지 객체
- **명시적 인텐트**: 호출할 액티비티 이름을 명확히 지정

```kotlin
val intent = Intent(applicationContext, SecondActivity::class.java)
startActivity(intent)
```

#### 단방향 데이터 전달 (putExtra / getExtra)

```
메인 액티비티                    세컨드 액티비티
    ①putExtra()  → startActivity() →  ③getExtra()
                                        ④finish()
```

- **보내기**: `intent.putExtra("키", 값)`
- **받기**: `intent.getStringExtra("키")`, `intent.getIntExtra("키", 기본값)`, `intent.getBooleanExtra("키", 기본값)`

#### 레이팅바 (RatingBar)

```xml
<RatingBar android:id="@+id/ratingBar1" />
<RatingBar
    android:id="@+id/ratingBar2"
    style="?android:attr/ratingBarStyleSmall"
    android:stepSize="1"
    android:numStars="10" />
<RatingBar
    android:id="@+id/ratingBar3"
    android:rating="1.5"
    style="?android:attr/ratingBarStyleIndicator" />
```

## Section 02. 액티비티와 인텐트의 응용

### 01 양방향 액티비티

메인 ↔ 세컨드 액티비티 간 양방향 데이터 전달

```
메인 액티비티                         세컨드 액티비티
①putExtra()                          ③getExtra()
②startActivityForResult()  →  데이터 →
⑦getExtra()               ←  데이터 ←  ④putExtra()
  onActivityResult()                  ⑤setResult()
                                      ⑥finish()
```

**메인 액티비티 코드:**
```kotlin
// 세컨드 액티비티 호출
val intent = Intent(applicationContext, SecondActivity::class.java)
intent.putExtra("Num1", Integer.parseInt(edtNum1.text.toString()))
intent.putExtra("Num2", Integer.parseInt(edtNum2.text.toString()))
startActivityForResult(intent, 0)

// 결과 받기
override fun onActivityResult(requestCode: Int, resultCode: Int, data: Intent?) {
    super.onActivityResult(requestCode, resultCode, data)
    if (resultCode == Activity.RESULT_OK) {
        val hap = data!!.getIntExtra("Hap", 0)
        Toast.makeText(applicationContext, "합계 : $hap", Toast.LENGTH_SHORT).show()
    }
}
```

**세컨드 액티비티 코드:**
```kotlin
val inIntent = intent
val hapValue = inIntent.getIntExtra("Num1", 0) + inIntent.getIntExtra("Num2", 0)

btnReturn.setOnClickListener {
    val outIntent = Intent(applicationContext, MainActivity::class.java)
    outIntent.putExtra("Hap", hapValue)
    setResult(Activity.RESULT_OK, outIntent)
    finish()
}
```

### 02 암시적 인텐트 (Implicit Intent)

약속된 액션을 지정하여 안드로이드 기본 앱을 실행

| 기능 | 코드 예시 |
|---|---|
| 전화 걸기 | `Intent(Intent.ACTION_DIAL, Uri.parse("tel:010-1234-5678"))` |
| 웹 브라우저 | `Intent(Intent.ACTION_VIEW, Uri.parse("http://www.example.com"))` |
| 구글 맵 | `Intent(Intent.ACTION_VIEW, Uri.parse("https://maps.google.com/maps?q=37.55,126.92"))` |
| 구글 검색 | `Intent(Intent.ACTION_WEB_SEARCH)` + `putExtra(SearchManager.QUERY, "검색어")` |
| 문자 보내기 | `Intent(Intent.ACTION_SENDTO)` + `putExtra("sms_body", "내용")` |
| 사진 찍기 | `Intent(MediaStore.ACTION_IMAGE_CAPTURE)` |

**권한 설정 (AndroidManifest.xml):**
```xml
<uses-permission android:name="android.permission.CALL_PHONE"/>
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION"/>
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>
```

### 03 액티비티 생명주기

```
        시작
         ↓
    onCreate()
         ↓
     onStart()  ←────────────── onRestart()
         ↓                            ↑
    onResume()               [다른 액티비티 종료]
         ↓
  [메인 액티비티 실행]
    ↙           ↘
[종료]       [다른 액티비티 요청]
  ↓                    ↓
onPause()          onPause()
  ↓                    ↓
onStop()           onStop() ──→ onRestart()
  ↓
onDestroy()
  ↓
 종료
```

| 메서드 | 호출 시점 |
|---|---|
| `onCreate()` | 액티비티 최초 생성 시 |
| `onStart()` | 화면에 보이기 시작할 때 |
| `onResume()` | 사용자와 상호작용 시작 |
| `onPause()` | 다른 액티비티로 이동할 때 |
| `onStop()` | 화면에서 완전히 사라질 때 |
| `onRestart()` | 중지 후 다시 시작할 때 |
| `onDestroy()` | 액티비티 완전히 종료 시 |

**로그캣 활용:**
```kotlin
android.util.Log.i("액티비티 테스트", "onCreate()")
android.util.Log.i("액티비티 테스트", "onDestroy()")
// ... 각 메서드에 로그 추가
```

## 핵심 정리

| 구분 | 설명 |
|---|---|
| **명시적 인텐트** | 액티비티 이름을 직접 지정 (`SecondActivity::class.java`) |
| **암시적 인텐트** | 액션 + URI로 시스템 앱 실행 (`Intent.ACTION_DIAL` 등) |
| **단방향 전달** | `putExtra()` → `startActivity()` → `getExtra()` |
| **양방향 전달** | `startActivityForResult()` → `setResult()` → `onActivityResult()` |
| **매니페스트 등록** | 추가한 모든 액티비티는 `AndroidManifest.xml`에 등록 필수 |
