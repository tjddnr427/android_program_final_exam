## 03-1 함수 선언하고 호출하기

### 함수의 기본 구조

```kotlin
fun 함수이름([변수이름: 자료형, 변수이름: 자료형 ...]): [반환값의 자료형] {
    표현식...
    [return 반환값]
}
```

```kotlin
fun sum(a: Int, b: Int): Int {
    var sum = a + b
    return sum
}
```

### 함수를 간략하게 표현하기

단계별로 생략 가능하다.

```kotlin
fun sum(a: Int, b: Int): Int { return a + b }   // 일반형
fun sum(a: Int, b: Int): Int = a + b            // 단일 표현식(=)으로 간략화
fun sum(a: Int, b: Int) = a + b                 // 반환 자료형 생략(추론)
```

```kotlin
// if 표현식을 단일 표현식 함수로
fun max(a: Int, b: Int) = if (a > b) a else b
```

### Unit 형 (반환값이 없는 함수)

반환값이 없으면 `Unit`을 쓰며, 생략 가능하다.

```kotlin
fun printSum(a: Int, b: Int): Unit {            // Unit 명시
    println("sum of $a and $b is ${a + b}")
}

fun printSum(a: Int, b: Int) {                  // Unit 생략
    println("sum of $a and $b is ${a + b}")
}
```

### 매개변수의 기본값 (Default Parameter)

매개변수에 기본값을 지정하면, 인자를 생략하고 호출할 수 있다.

```kotlin
fun add(name: String, email: String = "default") {
    // email로 넘어오는 값이 없으면 자동으로 "default" 사용
}

add("Youngdeok")    // email 생략 → "default" 사용
```

```kotlin
fun defaultArgs(x: Int = 100, y: Int = 200) {
    println(x + y)
}

defaultArgs()       // 100 + 200 = 300
defaultArgs(200)    // 200 + 200 = 400
```

### 이름 붙은 매개변수 (Named Parameter)

인자 이름을 지정해 호출하면 순서와 무관하게 전달할 수 있다. 기본값이 없는 매개변수가 뒤에 있어도 호출 가능.

```kotlin
fun namedParam(x: Int = 100, y: Int = 200, z: Int) {
    println(x + y + z)
}

namedParam(x = 200, z = 100)    // y는 기본값(200) 사용
namedParam(z = 150)             // x, y 모두 기본값 사용
```

### 가변 인자 (vararg)

매개변수 개수가 고정되지 않은 함수. `vararg` 키워드를 사용한다.

```kotlin
fun normalVarargs(vararg counts: Int) {
    for (num in counts) {
        println("$num")
    }
}

normalVarargs(1, 2, 3, 4)   // 4개 인자
normalVarargs(4, 5, 6)      // 3개 인자
```

## 03-2 함수형 프로그래밍

### 코틀린은 다중 패러다임 언어

- 함수형 프로그래밍(FP: Functional Programming)
- 객체 지향 프로그래밍(OOP: Object-Oriented Programming)

**함수형 프로그래밍의 특징**
- 코드 간략, 테스트·재사용성 증가
- 람다식, 고차 함수를 사용해 구성
- 순수 함수 사용

### 순수 함수 (Pure Function)

부작용(side-effect)이 없는 함수.

**조건**
1. 같은 인자에 대해 항상 같은 값을 반환 (결정적, deterministic)
2. 함수 외부의 어떤 상태도 바꾸지 않음

```kotlin
// 순수 함수의 예
fun sum(a: Int, b: Int): Int {
    return a + b    // 부작용 없음
}
```

**순수 함수가 아닌 예**

```kotlin
fun check() {
    val test = User.grade()           // 외부 User 객체에 의존
    if (test != null) process(test)   // 결과가 외부에 따라 달라짐
}

const val global = 10
fun noPureFunction(a: Int, b: Int): Int {
    return a + b + global             // 외부 변수 사용 → 순수 함수 아님
}
```

### 함수형 프로그래밍에 적용되는 개념

- 함수를 매개변수, 인자, 반환값에 사용 (고차 함수)
- 함수를 변수나 데이터 구조에 저장
- 유연성 증가

### 람다식 (Lambda Expression)

이름 없이 사용·실행 가능한 익명 함수의 한 형태. 람다 대수(Lambda calculus)에서 유래.

```kotlin
{ x, y -> x + y }   // 이름 없는 함수 형태
```

- 고차 함수에서 인자로 넘기거나 결과값으로 반환할 수 있다.

### 일급 객체 (First Class Citizen)

- 함수의 인자로 전달 가능
- 함수의 반환값으로 사용 가능
- 변수에 담을 수 있음

> 코틀린에서 함수는 1급 객체로 다루며, **1급 함수**라고도 한다.

### 고차 함수 (High-order Function)

함수를 인자나 반환값으로 사용하는 함수.

```kotlin
fun main() {
    println(highFunc({ x, y -> x + y }, 10, 20))  // 람다식을 인자로 전달
}

fun highFunc(sum: (Int, Int) -> Int, a: Int, b: Int): Int = sum(a, b)
```

### 정리

> 함수형 프로그래밍은 순수 함수를 조합해 상태 데이터 변경이나 부작용이 없는 루틴을 만들고, 이름 없는 함수 형태인 람다식을 이용해 함수를 변수처럼 매개변수·인자·반환값에 활용하는 고차 함수를 구성해 생산성을 높이는 프로그래밍 방법이다.

## 03-3 고차 함수와 람다식

### 일반 함수를 인자/반환값으로 사용

```kotlin
// 인자에 함수 사용
fun main() {
    val res1 = sum(3, 2)             // 일반 인자
    val res2 = mul(sum(3, 3), 3)     // 인자에 함수 사용
}
fun sum(a: Int, b: Int) = a + b
fun mul(a: Int, b: Int) = a * b
```

```kotlin
// 반환값에 함수 사용
fun funcFunc(): Int {
    return sum(2, 2)
}
```

### 변수에 할당하는 람다식

```kotlin
fun main() {
    var result: Int
    val multi = { x: Int, y: Int -> x * y }   // 람다식을 변수에 할당
    result = multi(10, 20)                    // 함수처럼 사용
    println(result)
}
```

### 람다식의 구성

```kotlin
val multi: (Int, Int) -> Int = { x: Int, y: Int -> x * y }
//         └─ 선언 자료형 ─┘     └─ 매개변수 ─┘    └ 처리내용 ┘
```

- **변수**: 함수처럼 사용 가능
- **선언 자료형**: 람다식 매개변수에 자료형이 명시된 경우 생략 가능
- **매개변수**: 선언 자료형이 명시되어 있으면 생략 가능
- **처리내용**: 표현식이 여러 줄이면 마지막 표현식이 반환됨

### 자료형 생략 규칙

```kotlin
val multi: (Int, Int) -> Int = { x: Int, y: Int -> x * y }  // 전체 표현
val multi = { x: Int, y: Int -> x * y }                     // 선언 자료형 생략
val multi: (Int, Int) -> Int = { x, y -> x * y }            // 매개변수 자료형 생략
val multi = { x, y -> x * y }                               // 에러! 추론 불가
```

여러 줄일 때:

```kotlin
val multi2: (Int, Int) -> Int = { x: Int, y: Int ->
    println("x * y")
    x * y   // 마지막 표현식이 반환됨
}
```

### 반환값이 없거나 매개변수가 하나일 때

```kotlin
val greet: () -> Unit = { println("Hello World!") }
val square: (Int) -> Int = { x -> x * x }
val nestedLambda: () -> () -> Unit = { { println("nested") } }
```

선언부 자료형 생략:

```kotlin
val greet = { println("Hello World!") }          // 추론 가능
val square = { x: Int -> x * x }                  // x의 자료형 명시 필요
val nestedLambda = { { println("nested") } }      // 추론 가능
```

### 매개변수로 람다식을 받는 고차 함수

```kotlin
fun main() {
    var result: Int
    result = highOrder({ x, y -> x + y }, 10, 20)
    println(result)
}

fun highOrder(sum: (Int, Int) -> Int, a: Int, b: Int): Int {
    return sum(a, b)
}
```

### 람다식을 변수에 저장 후 재사용

```kotlin
fun main() {
    val out: () -> Unit = { println("Hello World!") }
    out()                // 함수처럼 사용
    val new = out        // 변수처럼 할당
    new()                // 재사용
}
```

### 호출 방식

#### 값에 의한 호출 (Call by Value)

람다식 함수는 **값으로 처리**되어, 호출 시 즉시 수행된 후 결과값이 전달된다.

```kotlin
fun main() {
    val result = callByValue(lambda())   // 람다식 함수를 호출(즉시 실행)
    println(result)
}

fun callByValue(b: Boolean): Boolean {   // 일반 변수 자료형 매개변수
    println("callByValue function")
    return b
}

val lambda: () -> Boolean = {
    println("lambda function")
    true     // 마지막 표현식이 반환
}
```

#### 이름에 의한 호출 (Call by Name)

람다식 이름을 매개변수로 전달하면, **람다식 자체가 복사**되어 함수 내부에서 호출될 때 실행된다.

```kotlin
fun main() {
    val result = callByName(otherLambda)   // 람다식 이름으로 호출
    println(result)
}

fun callByName(b: () -> Boolean): Boolean {  // 람다식 함수 자료형 매개변수
    println("callByName function")
    return b()                               // 여기서 실행됨
}

val otherLambda: () -> Boolean = {
    println("otherLambda function")
    true
}
```

#### 함수 참조에 의한 호출 (`::`)

일반 함수를 인자로 넘길 때는 `::` (참조)를 사용한다.

```kotlin
funcParam(3, 2, sum)     // 오류! sum은 람다식이 아님
funcParam(3, 2, ::sum)   // OK! 함수 참조

fun funcParam(a: Int, b: Int, c: (Int, Int) -> Int): Int {
    return c(a, b)
}
fun sum(x: Int, y: Int) = x + y
```

활용 예:

```kotlin
fun main() {
    val res1 = funcParam(3, 2, ::sum)   // 인자·반환값 있는 함수
    hello(::text)                       // 인자 없는(반환 없는) 함수
    val likeLambda = ::sum              // 변수에 할당
    println(likeLambda(6, 6))
}
fun sum(a: Int, b: Int) = a + b
fun text(a: String, b: String) = "Hi! $a $b"
fun hello(body: (String, String) -> String): Unit {
    println(body("Hello", "World"))
}
```

### 람다식 함수의 매개변수 개수별 구성

```kotlin
// 매개변수가 없는 경우
noParam({ "Hello World!" })
noParam { "Hello World!" }              // 소괄호 생략 가능
fun noParam(out: () -> String) = println(out())
```

```kotlin
// 매개변수가 한 개인 경우
oneParam({ a -> "Hello World! $a" })
oneParam { a -> "Hello World! $a" }     // 소괄호 생략 가능
oneParam { "Hello World! $it" }         // it으로 대체 가능
fun oneParam(out: (String) -> String) { println(out("OneParam")) }
```

```kotlin
// 매개변수가 두 개 이상인 경우 → 매개변수명 생략 불가
moreParam { a, b -> "Hello World! $a $b" }
fun moreParam(out: (String, String) -> String) {
    println(out("OneParam", "TwoParam"))
}
```

```kotlin
// 일반 매개변수 + 람다식 매개변수 (마지막이 람다식이면 소괄호 밖으로 분리 가능)
withArgs("Arg1", "Arg2", { a, b -> "Hello World! $a $b" })   // ①
withArgs("Arg1", "Arg2") { a, b -> "Hello World! $a $b" }    // ②(동일)
fun withArgs(a: String, b: String, out: (String, String) -> String) {
    println(out(a, b))
}
```

```kotlin
// 두 개의 람다식을 가진 함수
twoLambda({ a, b -> "First $a $b" }, { "Second $it" })
twoLambda({ a, b -> "First $a $b" }) { "Second $it" }        // 동일
fun twoLambda(first: (String, String) -> String, second: (String) -> String) {
    println(first("OneParam", "TwoParam"))
    println(second("OneParam"))
}
```

호출 형태 정리:
```
({첫 번째}, {두 번째})
({첫 번째}) {두 번째}
({첫 번째}, {두 번째}) {세 번째}
```

## 03-4 / 03-5 코틀린의 다양한 함수들

### 익명 함수 (Anonymous Function)

이름이 없는 함수.

```kotlin
fun(x: Int, y: Int): Int = x + y                    // 익명 함수
val add: (Int, Int) -> Int = fun(x, y) = x + y      // 익명 함수로 add 선언
val add = fun(x: Int, y: Int) = x + y
val add = { x: Int, y: Int -> x + y }               // 람다식과 매우 흡사
```

> 일반 익명 함수에서는 `return`, `break`, `continue`를 사용할 수 있지만, 람다식에서는 사용하기 어렵다(라벨 표기법과 함께 써야 함).

### 인라인 함수 (inline)

함수가 호출되는 곳에 함수 내용을 모두 복사 → 분기 없이 처리 → 성능 증가.

```kotlin
fun main() {
    shortFunc(3) { println("First call: $it") }
    shortFunc(5) { println("Second call: $it") }
}

inline fun shortFunc(a: Int, out: (Int) -> Unit) {
    println("Before calling out()")
    out(a)
    println("After calling out()")
}
```

**단점**: 코드가 복사되므로 내용이 많은 함수에 쓰면 코드량이 늘어남.

#### noinline

일부 람다식만 인라인되지 않게 한다.

```kotlin
inline fun shortFunc(a: Int, noinline out: (Int) -> Unit) {
    println("Before calling out()")
    out(a)
    println("After calling out()")
}
```

#### 비지역 반환 (non-local return)

인라인 함수에 전달된 람다식 안의 `return`은 바깥 함수까지 빠져나간다.

```kotlin
fun main() {
    shortFunc(3) {
        println("First call: $it")
        return    // ① 의도하지 않은 반환 (main까지 종료)
    }
}

inline fun shortFunc(a: Int, out: (Int) -> Unit) {
    println("Before calling out()")
    out(a)
    println("After calling out()")   // ② 실행되지 않음
}
```

### 확장 함수 (Extension Function)

클래스의 멤버 함수를 외부에서 추가할 수 있다. (표준 라이브러리 수정 없이 확장)

```kotlin
fun 확장대상.함수명(매개변수, ...): 반환값 {
    ...
    return 값
}
```

```kotlin
fun main() {
    val source = "Hello World!"
    val target = "Kotlin"
    println(source.getLongString(target))
}

// String을 확장
fun String.getLongString(target: String): String =
    if (this.length > target.length) this else target
```

> `this`는 확장 대상 자리에 있던 객체(여기서는 source)를 가리킨다.

### 중위 함수 (infix)

점(`.`)과 소괄호를 생략하고 연산자처럼 직관적으로 호출하는 표현법.

**중위 함수의 조건**
1. 멤버 메서드 또는 확장 함수여야 함
2. 매개변수가 **하나**여야 함
3. `infix` 키워드로 정의

```kotlin
fun main() {
    // val multi = 3.multiply(10)   // 일반 표현법
    val multi = 3 multiply 10       // 중위 표현법
    println("multi: $multi")
}

infix fun Int.multiply(x: Int): Int {
    return this * x
}
```

### 재귀 함수와 꼬리 재귀 (tailrec)

**재귀(recursion)**: 자기 자신을 다시 호출.

**필수 조건**
- 무한 호출에 빠지지 않도록 탈출 조건을 둔다.
- 스택을 사용하므로 호출 횟수를 무리하게 많이 지정하지 않는다.
- 코드를 복잡하지 않게 한다.

> 무한 호출 시 **스택 오버플로(Stack Overflow)** 발생 가능.

#### 일반 재귀 factorial

```kotlin
fun factorial(n: Int): Long {
    return if (n == 1) n.toLong() else n * factorial(n - 1)
}
// factorial(4)
// 4 * factorial(3)
// 4 * (3 * factorial(2))
// 4 * (3 * (2 * factorial(1)))
// 4 * (3 * (2 * 1)) = 24
```

#### 꼬리 재귀 (tailrec) — 스택 오버플로 방지

코틀린 고유의 `tailrec` 키워드 사용. 스택을 사용하지 않도록 최적화.

```kotlin
tailrec fun factorial(n: Int, run: Int = 1): Long {
    return if (n == 1) run.toLong() else factorial(n - 1, run * n)
}
```

#### 피보나치 수열

0, 1로 시작하여 n번째와 n+1번째 수의 합이 n+2번째가 되는 수열 (0, 1, 1, 2, 3, 5, 8 …).

```kotlin
// 일반 재귀
fun fibonacci(n: Int, a: Long, b: Long): Long {
    return if (n == 0) b else fibonacci(n - 1, a + b, a)
}
```

```kotlin
// 꼬리 재귀 + BigInteger (큰 수 처리)
import java.math.BigInteger

fun main() {
    val n = 100
    val first = BigInteger("0")
    val second = BigInteger("1")
    println(fibonacci(n, first, second))
}

tailrec fun fibonacci(n: Int, a: BigInteger, b: BigInteger): BigInteger {
    return if (n == 0) a else fibonacci(n - 1, b, a + b)
}
```

## 03-6 함수와 변수의 범위 (Scope)

### 최상위 함수와 지역 함수

```kotlin
fun main() {                        // 최상위 레벨 함수
    fun secondFunc(a: Int) { ... }  // 지역 함수 선언

    userFunc(4)      // 최상위 함수 → 선언 위치 상관 없이 사용
    secondFunc(2)    // 지역 함수 → 선언부가 먼저 나와야 사용 가능
}

fun userFunc(counts: Int) { ... }   // 사용자 정의 최상위 함수
```

**사용 범위 규칙**
- 최상위 함수: `main()` 전·후 어디에 선언해도 코드 내에서 호출 가능
- 지역 함수: 먼저 선언되어야 호출 가능

```kotlin
fun a() = b()             // 최상위 → b() 위치 무관하게 사용 가능
fun b() = println("b")

fun c() {
    fun d() = e()         // 오류! d는 지역함수, 아직 e()를 모름
    fun e() = println("e")
}

fun main() {
    a()                   // OK
    e()                   // 오류! e()는 c의 블록 밖에서 사용 불가
}
```

### 전역 변수와 지역 변수

**전역 변수 (Global)**
- 최상위에 선언, 프로그램 실행 동안 메모리에 유지(삭제 안 됨)
- 여러 요소가 동시에 접근하면 잘못된 동작 유발 가능
- 자주 안 쓰이면 메모리 자원 낭비

**지역 변수 (Local)**
- 특정 코드 블록 내에서만 사용
- 블록 시작 시 임시로 사용되며 주로 스택 메모리 사용

```kotlin
var global = 10                  // 전역 - 패키지 전체에 적용

fun main() {
    val local1 = 20              // main 블록 내에서만 유지
    val local2 = 21

    fun nestedFunc() {
        global += 1
        val local1 = 30          // 이 블록 내에서만 유지 (바깥 local1을 가림)
        println("nestedFunc local1: $local1")   // 30
        println("nestedFunc local2: $local2")   // 바깥 main의 local2 사용
        println("nestedFunc global: $global")
    }

    nestedFunc()
    outsideFunc()

    println("main global: $global")
    println("main local1: $local1")   // 20 (가려지지 않은 원래 값)
    println("main local2: $local2")
}

fun outsideFunc() {
    global += 1
    val outVal = "outside"
    println("outsideFunc global: $global")
    println("outsideFunc outVal: $outVal")
}
```

> 안쪽 블록에서 같은 이름의 변수를 선언하면 바깥 변수가 **가려진다(shadowing)**. 선언 안 한 변수는 바로 바깥 스코프의 것을 사용한다.
